# OfficeMate 项目代码导读

这个项目是一个基于 `Streamlit + LangChain + Chroma + OpenAI-compatible 模型接口` 的企业知识问答系统。整体上可以把代码分成 6 层来理解。

## 1. 入口层

- [app.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/app.py:1)
- [pages/1_知识上传.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/pages/1_知识上传.py:1)
- [pages/2_知识管理.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/pages/2_知识管理.py:1)

这一层只负责启动 Streamlit 页面，本身几乎不写业务逻辑。

## 2. 页面层

- [services/ui_pages.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/ui_pages.py:1)

页面层负责接收用户输入、展示结果、调用 service 层，不直接处理底层检索和模型细节。可以把它理解成“控制器”。

## 3. 文档入库层

- [services/document_service.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/document_service.py:1)
- [services/document_parser.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/document_parser.py:1)
- [services/vector_store.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/vector_store.py:1)
- [services/storage_service.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/storage_service.py:1)

这里负责把文档变成知识库，主流程是：

1. 读取上传文件
2. 解析为纯文本
3. 切片
4. 生成 embedding
5. 写入 Chroma
6. 把元数据和日志写入本地 JSON

## 4. 主 RAG 问答层

- [services/rag/chat_service.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/rag/chat_service.py:1)
- [services/rag/query.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/rag/query.py:1)
- [services/rag/planning.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/rag/planning.py:1)
- [services/rag/retrieval.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/rag/retrieval.py:1)
- [services/rag/selection.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/rag/selection.py:1)
- [services/rag/answering.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/rag/answering.py:1)

推荐阅读顺序：

1. `chat_service.py`
2. `query.py`
3. `planning.py`
4. `retrieval.py`
5. `selection.py`
6. `answering.py`

主流程可以概括为：

1. 判断问题类型
2. 改写问题并扩展检索词
3. 把复杂问题拆成子任务
4. 做向量检索和 BM25 检索
5. 用 rerank 和规则 bonus 挑出更好的证据
6. 针对子任务分别生成答案
7. 如果是多任务，再汇总成一份最终答案
8. 补上引用文档并记录日志

## 5. 实验型问答链路

- [decision_react/service.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/decision_react/service.py:1)
- [agent_react_rag/service.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/agent_react_rag/service.py:1)

这两套不是主页面默认链路，而是实验性对比方案：

- `decision_react`
  先让一个 Decision Agent 判断问题复杂度，再决定是否改写、拆任务、汇总

- `agent_react_rag`
  让真正的 `create_agent(...)` 自己决定何时调用工具

如果你是新手，建议先读主 RAG 链路，再读这两套变体。
