# OfficeMate 项目代码导读

这份文档面向刚接触这个项目的新手，帮助你快速建立“现在这套系统到底怎么组织”的整体认知。

当前主版本已经是：

- 前端：`Vue 3 + Vite`
- 后端：`FastAPI`
- 关系型数据：`MySQL`
- 向量数据：`Milvus`

仓库里还保留了旧版 `Streamlit` 代码，但它不再是主入口。你可以把旧代码理解成历史版本或对照材料。

## 1. 先看主入口

如果你只想先把项目跑起来，最先关注这几个文件：

- [main.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/main.py:1)
- [api/app.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/api/app.py:1)
- [frontend/src/App.vue](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/frontend/src/App.vue:1)

它们分别负责：

- `main.py`
  FastAPI 启动入口

- `api/app.py`
  后端所有核心路由，例如文档上传、问答、测评、任务查询

- `frontend/src/App.vue`
  当前 Vue 控制台的主要页面逻辑

## 2. 后端分层怎么看

当前后端大致可以分成 6 层来理解。

### 2.1 API 路由层

- [api/app.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/api/app.py:1)
- [api/helpers.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/api/helpers.py:1)
- [api/schemas.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/api/schemas.py:1)

这一层负责：

- 接收 HTTP 请求
- 校验请求参数
- 调用 service 层
- 返回 JSON / SSE 响应

如果你想知道某个按钮最后调用了哪个接口，通常先从这里找。

### 2.2 基础设施层

- [core/bootstrap.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/core/bootstrap.py:1)
- [core/db.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/core/db.py:1)

这一层负责：

- 启动时建表或初始化环境
- 初始化数据库连接
- 保证 MySQL 和 Milvus 相关依赖能被业务层使用

### 2.3 ORM 模型层

- [models/entities.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/models/entities.py:1)

这一层定义 MySQL 里的核心表，例如：

- `documents`
- `qa_logs`
- `feedback_logs`
- `task_runs`
- `benchmark_runs`
- `benchmark_run_details`

如果你想知道数据最终是怎么落库的，这里是最直接的入口。

### 2.4 Service 业务层

这是项目的核心层，主要文件包括：

- [services/document_service.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/document_service.py:1)
- [services/chat_service.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/chat_service.py:1)
- [services/storage_service.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/storage_service.py:1)

- [services/benchmark_eval_service.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/benchmark_eval_service.py:1)
- [services/local_eval_service.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/services/local_eval_service.py:1)

它们大致分工如下：

- `document_service`
  文档上传、解析、入库、删除

- `chat_service`
  标准 RAG 问答编排

- `storage_service`
  文档、日志、反馈、测评结果等持久化操作

- `task_run_service`
  长任务状态管理

- `benchmark_eval_service`
  RAGBench 测评逻辑

- `local_eval_service`

2. `chat_service.py`
3. `query.py`

6. `selection.py`

4. 检索证据
5. 重排证据
6. 生成子任务答案
7. 汇总最终答案

实际主链路是：

- 前端发起问答
- 后端接收请求
- 标准 RAG 或 Agent 链路执行
- 后端把状态和结果返回前端
- 前端展示回答、思考过程、会话历史

## 5. Agent 链路怎么看

项目中保留了两套 Agent 化方案：

- [decision_react/service.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/decision_react/service.py:1)
- [agent_react_rag/service.py](/Users/weijiaxin/Documents/pythonwork/OfficeMate-main/OfficeMate-main/agent_react_rag/service.py:1)

你可以这样理解：

- `decision_react`
  先决策，再决定走哪些步骤

- `agent_react_rag`
  把 RAG 能力封装成工具，让 Agent 自己调用

如果你是新手，建议先读标准 RAG，再读 Agent。因为 Agent 其实是建立在你已经理解 RAG 主链路的基础上。
